The Journey of Crawling Medium API Data and Battling Cloudflare Defenses

Using Medium GraphQL Private API to Scrape Data, Backup Articles, and Bypass Cloudflare Verification Blocks.

Background

Since I started writing on Medium in 2018, by around 2022 I had accumulated over 50 articles, nearly 60,000 words, and about 3 GB of article images (by 2025, over 120 articles/100,000+ words/6 GB+ images); no backups! no backups! no backups! Because I always wrote directly on Medium (the editor is really good), I published immediately after writing without any local backups.

At that time, a sense of unease arose: what if Medium shut down one day and my articles disappeared? (And Medium had been struggling for years, not making money) Or what if my account suddenly got suspended and all my hard work was lost?

So that year, I started researching how to scrape Medium data, especially article content and how to download backups.

Medium HTML ❌

The structure is very complex, prone to changes, and slow to crawl. Directly using a browser to scrape content is the worst approach.

Medium Official API ❌

Warning The Medium API is no longer supported. We do not recommend using it.

Medium once provided an official API, but it was likely discontinued shortly after launch. It is assumed this was due to different product goals—Medium aims to keep content within its own platform rather than allowing external access through an API.

Medium Unofficial API 🤔

Since the official API is not provided, some developers have created external API interfaces for users in need; however, I suspect the underlying principle is still calling the Medium Private API, just with an additional layer of encapsulation that converts your requests into Private API calls to fetch the original data.

The fee is not cheap, about $USD 5 for every 2,500 requests ; this service has been running for quite a while, but since it ultimately calls the Medium Private API, I might as well sniff and call it myself.

Medium Private API ✅

What is a Private API? It is an API interface not intended for external use, undocumented, may be disabled at any time, and may carry legal risks; basically, directly observing the website/App behavior to find the corresponding API calls and writing your own code to reuse this API to fetch data for your own service.

https://medium.com/_/graphql

By sniffing network requests, we can find that all Medium APIs use the https://medium.com/_/graphql endpoint. As long as we can call this in our own program and get the same data, this approach will work.

Medium Private API — Sniffing Medium GraphQL (Example: Getting Follower Count)

My first application is to crawl my follower count and display it on my portal site.

link.zhgchg.li :

Therefore, I must scrape the follower count of my account on Medium.

  • First, open the browser’s “Developer Mode” (Chrome -> Right-click -> Inspect), switch to the “Network” tab, check “Preserve log,” and enter “graphql” in the search box.

  • Go to the Medium account followers page: https://medium.com/@YOUR_USER_NAME/followers
    Only this format of follower URL can be sniffed; https://YOUR_USER_NAME.medium.com/followers does not call the Followers API.

  • Check each “graphql” request on the left, search for the keyword “followerCount” in the “Response” data, and find the request where the username matches the webpage’s username and the follower count matches the number shown on the page. This is the target request.

  • After finding the target API, switch to Payload -> View Source -> copy all the following content.
